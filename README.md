# Hybrid Recommendation System
**iCog Labs Internship Training Task**

## Overview
A hybrid product recommendation system built on the Amazon Electronics dataset. 
Combines collaborative filtering and content-based filtering to address four 
core recommender system problems: cold start, matrix sparsity, popularity bias, 
and filter bubble.

## Architecture

This system uses a **cascade hybrid** architecture where each component has a 
clearly defined role:

| Component | Method | Purpose |
|---|---|---|
| Collaborative Filtering | Surprise SVD (20 latent factors) | Warm user recommendations |
| Content-Based Filtering | TF-IDF (100 features) on product metadata | Cold item representation |
| Cold User Fallback | Global popularity ranking | New user onboarding |
| Re-ranking | Maximal Marginal Relevance (λ=0.5) | Diversity, filter bubble reduction |

## Problems Addressed

**Cold Start** — New users receive popularity-based fallback recommendations. 
New items are represented via TF-IDF metadata embeddings so they can be 
recommended without interaction history.

**Matrix Sparsity** — Dataset sparsity is 99.99%. Addressed by filtering users 
and items below 5 interactions and applying SVD matrix factorization with 
regularization to learn dense latent factor representations.

**Popularity Bias** — Monitored via Coverage metric. MMR re-ranking penalizes 
homogeneous recommendations to surface niche items.

**Filter Bubble** — Addressed with MMR (λ=0.5), balancing relevance and 
diversity. Measured using Intra-List Diversity (ILD).

## Dataset
- **Ratings**: Amazon Electronics (Kaggle) — 7.8M interactions, filtered to 1.7M
- **Metadata**: Amazon meta_Electronics.json.gz — product titles, categories, descriptions
- **Final training set**: ~644K interactions across 161K users and 36K items

## Architectural Decisions

**Why cascade hybrid over joint training?**  
LightFM (joint training) encountered Windows/OpenMP compilation issues. The 
cascade approach — SVD for CF, TF-IDF independently for content — is one of 
the four hybrid strategies in the reference material and offers clearer 
interpretability: each component has an explicit, auditable role.

**Why Surprise SVD over other CF methods?**  
Pure Python/NumPy implementation with no platform-specific compilation. 
Mathematically equivalent to standard matrix factorization: decomposes the 
user-item matrix into user latent factors P and item latent factors Q, 
computing relevance as r = P^T · Q.

**Why λ=0.5 for MMR?**  
Equal weighting between relevance and diversity. This was chosen empirically — 
higher λ increases precision but reduces ILD gains; lower λ maximises diversity 
but risks irrelevant recommendations.
